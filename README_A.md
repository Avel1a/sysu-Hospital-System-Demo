基于 Python Streamlit 和 SQLite 开发的轻量级门诊管理系统。本项目实现了患者预约、前台分诊、收费结算及医院行政管理的完整业务闭环。

核心亮点：

 无代码前端：使用 Streamlit 快速构建响应式 Web 界面。

 自动化数据库：启动时自动加载 schema.sql，利用 SQL 触发器实现业务状态自动流转。

 数据可视化：自动生成科室收入统计图表。

 演示模式：内置“一键生成/清空演示数据”功能，专为答辩演示设计。

 功能模块
1.  患者端
自助预约：填写姓名、科室、电话及预计到达时间。

状态查询：通过电话号码实时查询就诊进度（待就诊/就诊中/已离院）及分配诊室。

2.  前台端
预约核验：自动拉取待就诊列表，一键填充模拟身份证号，分配诊室并转入就诊序列。

收费结算：支持医保/自费金额自动拆分。亮点： 缴费成功后，数据库触发器自动将患者状态更新为“已离院”。

全院查询：全局搜索患者信息，实时查看全院就诊状态。

3.  管理员端 
数据看板：按科室统计门诊收入，基于 SQL 视图 实现。

员工管理：

查看全院员工列表。

新增员工：录入医生/护士信息。

修改信息：支持对员工职称、排班时间、工作状态等进行编辑回显与保存。

 本地部署与运行指南
环境要求
Python 3.8 或更高版本

依赖库：streamlit, pandas

步骤 1：克隆项目
Bash

git clone https://github.com/你的用户名/你的项目名.git
cd 你的项目名
步骤 2：安装依赖
建议使用虚拟环境，或者直接安装：

Bash

pip install streamlit pandas


步骤 3：检查关键文件
请确保项目根目录下存在 schema.sql 文件。

系统启动时会强制读取该文件来初始化数据库结构（包含建表、视图和触发器）。

注意：hospital.db 文件是自动生成的，无需手动创建。

步骤 4：启动系统
在终端运行以下命令：

Bash

streamlit run app.py
启动成功后，浏览器会自动打开 http://localhost:8501
